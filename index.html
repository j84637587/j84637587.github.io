發表於 2026-02-26 更新於 2026-03-10

深度學習模型訓練方法整理：DDP、DP 與 Gradient Accumulation

在實務訓練深度學習模型時，單張 GPU 往往無法滿足速度或記憶體需求。因此我們會透過平行化訓練與梯度技巧來優化效率與穩定性。

這是最原始的訓練方法，資料會先被分成多個 mini-batch，訓練後更新模型權重，以此反覆。

本文整理三種常見方法：

  • DDP（Distributed Data Parallel）
  • DP（Data Parallel）
  • Gradient Accumulation

一、DDP（Distributed Data Parallel）

概念

DDP 是目前主流的多 GPU 訓練方式，廣泛使用於大型模型訓練中。
在很多大型模型或是資料集非常大量的時候非常有用(節省n個GPU的訓練時間)。

運作原理是建立多個Process，每個Process對應一張GPU，並且在每個Process中複製一份模型。每個Process負責處理不同的資料子集，並且在每次反向傳播後使用All-Reduce來同步梯度，確保所有模型複本的權重保持一致。

這種方式可以跨主機訓練，適合大規模分布式訓練。

題外話: 研究生時期瀏覽了很多開源模型，發現有些專案雖然表明使用DDP但實際上是使用DP，這種情形會導致訓練時雖然主機有N張GPU但實際上只有1張GPU在訓練。

運作方式

  1. 在每個 GPU 上 複製一份模型
  2. 將資料分成不同 mini-batch 分配給各 GPU
  3. 每個 GPU 各自計算梯度
  4. 使用 All-Reduce 同步梯度
  5. 同步更新全部模型複本，保持所有模型起始訓練時的權重一致

梯度是如何跨主機同步的？

DDP 使用了分布式通信協議(NCCL、Gloo)來實現跨主機的梯度同步，這是一種專為訓練設計的Socket技術(就是網路通訊)。

因為NCCL或Gloo會開啟Port來傳輸資料，因此有時候發生NCCL或Gloo的Port被佔用的情況，就會發生訓練錯誤。

二、DP（Data Parallel）

概念

DP 也是常見的訓練方式。與 DDP 不同的是，DP 限制在單主機內的多 GPU 訓練，無法跨主機訓練。

運作方式

  1. 在多個 GPU 上複製相同模型
  2. 將資料分成多個 mini-batch 分配給各 GPU
  3. 每個 GPU 各自計算梯度
  4. 主 GPU 收集所有 GPU 的梯度，進行平均
  5. 將梯度彙總到主 GPU 統一更新

DP 的實作相對簡單，但由於資料要先彙整到主GPU，因此主 GPU 可能成為瓶頸。

三、Gradient Accumulation

概念

當 GPU 記憶體不足以支撐大 batch size 時，可以使用梯度累積技巧。

運作方式

  • 將一個大 batch 拆成多個小 batch
  • 每次 forward/backward 只計算梯度
  • 不立即更新權重
  • 累積指定步數後再更新一次

為什麼有效？

大 batch size 通常有：

  • 更穩定的梯度
  • 較低的梯度噪聲
  • 更好的收斂特性

Gradient Accumulation 可以：

  • 在不增加顯存的情況下
  • 模擬更大的 batch size 效果

LLM 訓練需要的資料量非常大，通常只訓練 1 Epoch 就需要幾十萬甚至幾百萬的 batch size，因此梯度累積是必須的。

方法比較總整理

方法 適用情境 訓練速度 實作難度
DDP 模型大 + 資料大 超快 最難
DP 模型小 + 資料大 快 簡單(一行python就可以)
Gradient Accumulation 顯存不足 不提升速度 簡單

2026-02-04-梯度是甚麼?

發表於 2026-02-04 更新於 2026-03-10

梯度是甚麼?

一句話解釋：
梯度就是每個神經元(參數)往"哪個方向調整"以及"要調整的幅度是多少"會讓損失函數增加最快？

在深度學習中，通常會沿著梯度的反方向更新參數，因為梯度指向損失函數增加最快的方向，所以我們要往相反的方向走，才能讓損失函數下降。

梯度（Gradient）雖然聽起來很專業，但其實就是我們在國中時期(應該是?)學到的斜率、速率的概念。在深度學習中，梯度用來指示損失函數(Loss function)相對於模型參數的變化率。

阿嬤教你梯度

阿嬤正走去早上的菜市場，阿嬤距離菜市場100公尺，花了20秒走到，假設阿嬤行走的速度不變，阿嬤的速度(速率)是多少？

上面的題目我們很容易的可以得到式子：

速度=100/20=5(公尺/每秒)速度 = 100 / 20 = 5(公尺/每秒) 速度=100/20=5(公尺/每秒)

速度 5(公尺/每秒) 的計算過程，類似於我們在深度學習中計算變化率的概念。
嚴謹地說：速度=距離對時間的變化率
所以梯度，就是變化率。

換到深度學習的領域就是：梯度=損失函數對參數的變化率
只是在深度學習中，我們的變數(參數)不只有一個，用阿嬤舉例就是：阿嬤可以同時往東西南北走、有上坡、下坡，梯度就會是一個向量，每個方向都有速度。

這個向量就類似這張表：

方向 東 西 南 北 上坡 下坡
速度(公尺/每秒) 5 -3 0 2 1 -5

[進階1]數學表示

延續上面的題目，我們來找尋阿嬤用怎樣的速度才能在最短的時間抵達菜市場？

假設阿嬤的行走速度為 vvv，距離菜市場的距離為 ddd，那么花費的時間 ttt 可以表示為：

t=dvt = \frac{d}{v} t=vd​

我們要讓花費時間最少，就可以定義Loss為：

Loss(v)=花費時間=dvLoss(v) = 花費時間 = \frac{d}{v} Loss(v)=花費時間=vd​

梯度=損失函數對參數的變化率定義為：

符號 ∇\nabla∇ 代表梯度運算子/變化率/向量微分運算子

梯度=∇Loss(v)=∂Loss∂v=∂∂v(dv)=−dv2\text{梯度} = \nabla Loss(v) = \frac{\partial Loss}{\partial v} = \frac{\partial }{\partial v} \left( \frac{d}{v} \right) = -\frac{d}{v^2} 梯度=∇Loss(v)=∂v∂Loss​=∂v∂​(vd​)=−v2d​

帶入剛剛阿嬤的速度 5(公尺/每秒)以及距離 100 公尺：

∇Loss(v)=−10052=−10025=−4\nabla Loss(v) = -\frac{100}{5^2} = -\frac{100}{25} = -4 ∇Loss(v)=−52100​=−25100​=−4

這個 -4 告訴我們，當阿嬤以 5(公尺/每秒) 的速度行走時，如果她想要減少花費的時間，她應該增加她的速度。

所以就跟前面提到的，梯度指向損失函數增加最快的方向，因此沿著負梯度方向可以最快地降低損失。

實際上，在更新時還會乘上一個學習率(learning rate) η\etaη，來控制每次更新的幅度：

vnew=vold−η⋅∇Loss(v)v_{new} = v_{old} - \eta \cdot \nabla Loss(v) vnew​=vold​−η⋅∇Loss(v)

以上就是梯度的基本概念與數學表示。

[進階2]反向傳播（Backpropagation）

剛剛的阿嬤例子是單一參數的情況，在深度學習中，我們通常有成千上萬個參數需要更新，可以想像成是跑一個接力賽，每個跑者都需要知道自己應該往哪個方向跑，跑多快，這時候參數就會變多，這時候我們就需要用到反向傳播來計算這些參數的梯度。

由於每個參數會影響著下個參數，所以我們需要從最後一層開始，逐層往前計算每個參數的梯度，這樣才能確保每個參數都能正確地更新，而這就是反向傳播。

實際範例

假設一場 3000 公尺(ddd)接力賽，有 2 個跑者，每人跑 d2=1500\frac{d}{2} = 15002d​=1500 公尺：

  • 第一個跑者速度為 p1p_1p1​
  • 第二個跑者速度 p2=p1+2p_2 = p_1 + 2p2​=p1​+2（比第一個跑者快 2 公尺/秒）

目標函數(Loss)是總時間：

Loss(p1)=d2p1+d2p2=d2p1+d2(p1+2)Loss(p_1) = \frac{d}{2p_1} + \frac{d}{2p_2} = \frac{d}{2p_1} + \frac{d}{2(p_1+2)} Loss(p1​)=2p1​d​+2p2​d​=2p1​d​+2(p1​+2)d​

第二個跑者的梯度

最簡單的是第二個跑者的梯度(最後一個輸出層)：

d2p2\frac{d}{2p_2}2p2​d​ 只受到 p2p_2p2​ 影響，所以直接對 p2p_2p2​ 求導數，d2p1\frac{d}{2p_1}2p1​d​ 視為常數，直接消去。

∂Loss∂p2=∂∂p2(d2p2)=−d2p22\frac{\partial Loss}{\partial p_2} = \frac{\partial }{\partial p_2}\left(\frac{d}{2p_2}\right) = -\frac{d}{2p_2^2} ∂p2​∂Loss​=∂p2​∂​(2p2​d​)=−2p22​d​

第一個跑者的梯度

計算第一個跑者的梯度就比較難了，因為計算 ∂Loss∂p1\frac{\partial Loss}{\partial p_1}∂p1​∂Loss​ 時，需要考慮兩種影響：

  1. 直接影響(p1p_1p1​ 直接影響自己跑步時間)，也就是第一項：

∂∂p1(d2p1)=−d2p12\frac{\partial}{\partial p_1}\left(\frac{d}{2p_1}\right) = -\frac{d}{2p_1^2} ∂p1​∂​(2p1​d​)=−2p12​d​

  1. 間接影響(p1p_1p1​ 影響第二個跑者時間)，第二項：

這裡需要用到連鎖律(chain rule)，否則是沒辦法直接計算Loss對p1p_1p1​的變化率。
∂Loss∂p2⋅∂p2∂p1\frac{\partial Loss}{\partial p_2} \cdot \frac{\partial p_2}{\partial p_1}∂p2​∂Loss​⋅∂p1​∂p2​​ 可以理解成：Loss對p2p_2p2​的變化率，乘上p2p_2p2​對p1p_1p1​的變化率，就會是Loss對p1p_1p1​的變化率。

∂∂p1(d2p2)=∂Loss∂p2⋅∂p2∂p1=−d2p22⋅1=−d2p22\frac{\partial}{\partial p_1}\left(\frac{d}{2p_2}\right) = \frac{\partial Loss}{\partial p_2} \cdot \frac{\partial p_2}{\partial p_1} = -\frac{d}{2p_2^2} \cdot 1 = -\frac{d}{2p_2^2} ∂p1​∂​(2p2​d​)=∂p2​∂Loss​⋅∂p1​∂p2​​=−2p22​d​⋅1=−2p22​d​

總梯度

∂Loss∂p1=−d2p12−d2p22\frac{\partial Loss}{\partial p_1} = -\frac{d}{2p_1^2} - \frac{d}{2p_2^2} ∂p1​∂Loss​=−2p12​d​−2p22​d​

這個例子說明了反向傳播的核心：當參數之間有依賴關係時，必須從後往前計算，把所有影響路徑的梯度加總起來。在多層神經網絡中，這個過程會逐層往前傳遞，直到第一層為止。

2025-07-20-Java Maven Protobuf 版本問題

發表於 2025-07-20 更新於 2026-03-10

今天在把自己做的 Java 專案從 Structured Packet 改為使用 Protobuf，但在使用時出現了一個錯誤：

一開始以為是套件版本對不上，即產生Protobuf類別檔案使用的執行檔與protobuf-java Maven Dependence，兩者版本對不上。

我使用的Protobuf執行檔最新版本是 v31.1 而protobuf-java是 4.28.3，看不出兩者的關聯。

在查詢許多文章、Git Issue後，發現的確是版本不對導致的問題。

跑去看了protobuf-java全部的版本後才知道，原來 v31.1 版本對應的protobuf-java版本就會是 X.31.1。

最後 protobuf-java 更新成 4.31.1 版本後就可以正常編譯了。

ROC曲線詳解-機器學習模型評估的重要指標

發表於 2025-07-09 更新於 2026-03-10

ROC 指標介紹

圖片來源：https://www.nomidl.com/machine-learning/what-is-the-roc-curve/

ROC（Receiver Operating Characteristic）曲線是一種用於評估分類模型效能的指標，特別適用於 二分類問題。

ROC 曲線的橫軸是FPR（False Positive Rate），縱軸是TPR（True Positive Rate），也稱為召回率（Recall）。

  • TPR（真正率）= TP / (TP + FN)：模型成功預測為正例的比例。
  • FPR（假陽性率）= FP / (FP + TN)：實際為負例卻被錯誤預測為正例的比例。

ROC 曲線呈現模型在不同閾值（threshold）下的分類表現，曲線越靠近左上角，表示模型效能越好。

假設今天我們訓練了一個模型判斷一張圖片是不是貓(1)，然後用 10 張圖片測試他，並用測試的結果計算 ROC 曲線，結果如下表。

1x1 卷積層介紹

發表於 2025-05-01 更新於 2026-03-10

什麼是 1x1 卷積層？

1x1 卷積層是一種特殊的卷積操作，其中卷積核的尺寸為 1x1。這意味著每次卷積操作只會影響輸入特徵圖中的單個像素，但會跨通道進行計算。

1x1 卷積層的用途

1x1 卷積層在深度學習中有以下幾個主要用途：

  1. 降維或升維：通過調整輸出通道數，1x1 卷積層可以用於壓縮或擴展特徵圖的通道數，從而減少計算量或增加模型表達能力。
  2. 非線性映射：1x1 卷積層可以引入非線性激活函數（如 ReLU），從而增加模型的非線性表達能力。
  3. 特徵融合：通過跨通道的加權求和，1x1 卷積層可以實現不同通道特徵的融合。

1x1 卷積層的數學表示

假設輸入特徵圖的尺寸為 ( H \times W \times C_{in} )，1x1 卷積層的輸出特徵圖尺寸為 ( H \times W \times C_{out} )。其計算公式為：

[
Y_{h,w,c} = \sum_{i=1}^{C_{in}} X_{h,w,i} \cdot W_{i,c} + b_c
]

其中：

  • Xh,w,iX_{h,w,i}Xh,w,i​ 是輸入特徵圖在位置 (h,w)(h, w)(h,w) 和第 iii 通道的值。
  • ( W_{i,c} ) 是 1x1 卷積核的權重。
  • ( b_c ) 是偏置項。
  • ( Y_{h,w,c} ) 是輸出特徵圖在位置 ( (h, w) ) 和通道 ( c ) 的值。

1x1 卷積層的優勢

  • 計算效率高：相比於更大的卷積核，1x1 卷積層的計算量更小。
  • 靈活性強：可以靈活調整通道數，適用於各種網絡結構。
  • 應用廣泛：在 GoogLeNet（Inception 模型）和 ResNet 等經典網絡中廣泛使用。

結論

1x1 卷積層是一種簡單但功能強大的卷積操作，能夠有效地提升模型的表達能力，同時降低計算成本。在設計卷積神經網絡時，合理使用 1x1 卷積層可以帶來顯著的性能提升。
